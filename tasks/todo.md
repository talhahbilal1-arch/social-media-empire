# Revenue Tracking & Analytics Dashboard

## Plan

Build 5 components for a revenue tracking and analytics system:

1. **`analytics/pin_tracker.py`** — Pin performance tracking script
   - Query Supabase `pinterest_pins` and `pinterest_analytics` tables
   - Aggregate pin impressions, saves, clicks by brand, style, topic
   - Identify top-performing pins and best posting patterns
   - Store weekly summary back to Supabase analytics table
   - Generate weekly report text file

2. **`analytics/article_tracker.py`** — Article performance script
   - Query `generated_articles` table for all articles
   - Scan article HTML files for affiliate link counts
   - Identify thin content (low word count, few links)
   - Suggest new topics based on top-performing pin topics
   - Output improvement recommendations

3. **`analytics/daily_report.py`** — Daily revenue report
   - Query Supabase for daily pin count, article count, errors
   - Calculate week-over-week growth percentages
   - Flag anomalies (drops >30%, zero-output days)
   - Output to ~/tall-command-center/briefings/

4. **`outputs/dashboard.html`** — Static HTML dashboard
   - Generated by `analytics/generate_dashboard.py`
   - Shows: pins per brand, articles per brand, top content, errors, deployment status
   - Clean, dark-themed design with brand colors

5. **`.github/workflows/daily-analytics.yml`** — GitHub Actions workflow
   - Run daily at 9 AM EST (14:00 UTC)
   - Execute pin_tracker, article_tracker, daily_report
   - Generate dashboard HTML
   - Commit updated dashboard
